# Blog Strategy Design

- **Date:** 2026-05-11
- **Status:** Draft — pending review
- **Branch:** `docs/blog-strategy`

## 목표

`jiyeon914.github.io` 블로그의 *방향성*을 명문화한다. 그동안 PR #1~#5는 사이트 *골격*(profile pages, 테마 정리, 에이전트 규칙, 레이아웃, README)을 만들었지만, "이 사이트에 *무엇을* / *왜* / *어떤 톤으로* 쓸 것인가"는 미정 상태였다. 이 문서는 그 질문에 답해서, 앞으로 6–12개월 동안 콘텐츠 작성·페이지 변경·소소한 follow-up 작업이 일관되게 한 방향을 가리키도록 한다.

이 spec은 *콘텐츠 전략 문서*이지 구현 플랜이 아니다. 구현은 다음 단계의 plan에서 다룬다.

## 합의된 결정 사항

| 항목 | 결정 |
|---|---|
| 사이트 목적 | 혼합 — 직업 포트폴리오 + 학습/연구 저널 + 자유로운 개인 공간. 한쪽에 강하게 베팅하지 않음 |
| 콘텐츠 종류 | 4가지 모두 — 기술 딥다이브 / 경력·워크 회고 / 읽고 배우는 것 노트 / 개인 글 |
| 빈도 | 2–3개월에 1편 (6개월 기준 2–3편). 저빈도·고밀도 |
| 톤 | 캐주얼·대화체 통일 (콘텐츠 종류 가변 X) |
| 구조 옵션 | A — 경량 태그 + 단일 chronological feed. 카테고리는 글 10편 누적 시점에 재검토 |
| Posts 영역 위치 | 기존 `/` (Minimal Mistakes 기본 홈 = 포스트 리스트) 그대로 사용 |

## 사이트 정체성 (Identity)

블로그는 *이중 구조*로 운영한다:

- **Profile 영역** (`/about/`, `/experience/`, `/research/`, `/publications/`): *공식 정보 앵커*. 정제된 어조. 한 번 잘 만들어두고 가끔만 업데이트 — 본인 정체성·이력·연구 영역을 외부 방문자에게 전달하는 역할.
- **Posts 영역** (`/`): *비공식·자유로운 voyage*. 캐주얼 통일톤. 4종 콘텐츠가 한 chronological feed에 섞여서 흐름. 글쓴이의 *목소리*가 사는 곳.

두 영역은 같은 사이트 안에 있지만 *역할이 다른* 게 디자인의 핵심이다. 한쪽이 다른 쪽을 침범하지 않게 운영한다. 예를 들어 *Experience* 페이지를 캐주얼하게 늘여 쓰지 않고, *Posts* 영역을 학술 톤으로 정제하지 않는다.

## Posts 영역 구조

### 피드

- 단일 chronological feed. 새 글이 위, 오래된 글이 아래
- 현재 홈(`/`)이 이미 포스트 리스트라 추가 페이지·UI 변경 없음
- 네비게이션의 *Posts* 탭이 `/`로 연결됨 — 그대로 유지

### 태그

- 미리 정의하지 않음
- 글을 쓸 때 자유롭게 1–3개 부여
- 같은 태그가 2–3회 누적되면 그게 자연 발생 카테고리
- 태그 페이지(`/tags/<tag>/`) 등 별도 UI는 *지금은 만들지 않음*. 태그 클릭으로 필터링이 필요한 시점이 오면 그때 한 줄 PR로 추가

### 포스트 frontmatter 표준

```yaml
---
title: "<제목>"
date: YYYY-MM-DD
tags: [tag1, tag2]
excerpt: "한두 줄 요약 — 홈 카드와 검색 결과에 표시됨"
---
```

- `layout`, `author_profile`은 `_config.yml`의 `defaults`(scope=`""`, type=`posts`)에서 이미 잡힘 — frontmatter에 반복 X
- `tags` 비워둬도 OK (저빈도라 강제 X)
- `excerpt`는 권장. 없으면 본문 앞 단락이 자동 사용됨

### 파일명 규칙

- `_posts/YYYY-MM-DD-slug.md` — Jekyll 기본
- slug는 영어 lowercase + hyphen (예: `2026-06-15-blog-setup-retrospective.md`). 한국어 slug는 Jekyll URL 인코딩이 어색해질 수 있어 피함

### 분기점

- 글 10편 누적 시점에 *카테고리 도입 여부* 재검토. 그 전까지는 태그만으로 운영.

## 콘텐츠 플랜

### 4 axis × 저빈도 페이스

저빈도(2–3편/6mo)에서 4가지 콘텐츠 종류를 *균등 분포* 시키려 하지 않는다. *가장 손이 가는 쪽*부터 자연 발생적으로 풀어내고, 결과적으로 6개월 후 어떤 mix가 되든 그게 본인 색깔이다.

### 첫 포스트 후보

순서·우선순위 없음. 어떤 걸 골라도 OK.

1. **블로그 만든 회고 (메타)** — Minimal Mistakes + Docker + Jekyll 셋업, AI 페어 프로그래밍하며 부딪힌 것들, PR #1~#5 시리즈 회고. 캐주얼 톤 잡기 좋고, 사이트 자체가 컨텍스트라 진입 장벽 낮음.
2. **PhD 졸업 + 산업계 전환 회고** — 학계→산업 이전 후 첫 몇 개월의 관찰. 진솔한 글, 미래 본인을 위한 기록.
3. **CFD + AI 도메인 한 줄 정리** — 본인 연구 영역을 비전공자도 읽을 수 있게 풀어 쓰기. "이 사이트가 누구의 사이트인지" 한 글로 보여주는 효과.
4. **최근 본 논문 / 책 1권 메모** — 가벼운 진입. 톤 연습용.

### 태그 시드 예시 (참고만, 미리 정의 X)

`blog-meta`, `career`, `cfd`, `deep-learning`, `reading`, `setup`, `reflection`, `kis`(자동매매), `paper-notes`, `tools` — 글에서 자연 발생할 만한 후보들. 처음부터 표준화하려 하지 않음.

## 톤·보이스 가이드

캐주얼·대화체 통일을 운영 차원으로 풀면:

- **1인칭 OK** ("저는…", "I…"). 객관체로 가지 않음
- **대화체 openers / asides OK**: "오랜만이에요", 괄호 부연, em-dash 활용 — 본인이 채팅할 때 톤이 baseline
- **한·영 코드스위칭 자연스럽게**: 기술 용어(CFD, FNO, diffusion model)는 영어, 일상 표현은 한국어. About 페이지 한국어 인사말 + 영어 bio 스타일이 기준
- **길이 유연**: 800~3000자 정도. 짧게 쓰고 싶은 날은 짧게. 강제 X
- **피할 것**:
  - 학술 3인칭, 건조한 abstract 스타일
  - 한 문단에 호흡 없는 긴 문장
  - "독자에게 가르치려는" 어조 — 사이트가 *튜토리얼*이 아님

## 프라이버시 자세

이미 `AGENTS.md` Hard Rules에 명시된 정책을 운영 차원으로 정리.

**공개 OK**

- 공개된 publications / conference 자료
- 직무 · 소속 (이미 About/Experience에 있음)
- 도메인 차원 논의 (CFD, turbulence, generative DL 등)
- 공개 논문 · 서적 리딩 노트
- 개인 에세이 (타인 프라이버시 침해 없으면)

**공개 ✗ (반드시 피함)**

- LG CNS NDA 정보: 고객사 데이터, 마케팅 자료에 없는 내부 아키텍처·코드네임, 미공개 프로젝트 상세
- 개인 식별: 집 주소, 전화번호, 가족 정보, 신분증 번호
- 구직 컨텍스트: 진행 중인 지원, 연봉 협상, 면접 후기 (PR #1 사고의 정책화)
- 타인 신원·험담: 동료/친구/지인 신상이나 부정적 평가
- (선택) 정치·종교적 강한 의견 — 직업 브랜드 관리 차원 본인 판단

**판단 기준 단순화**

> "6개월 뒤 다른 회사 채용 담당자가 봤을 때 본인을 어색하게 만들 만한 글인가?"

의심스러우면 publish 보류 → 비공개 draft(`_drafts/`)로 두기.

## 첫 6개월 로드맵

| 시점 | 마일스톤 |
|---|---|
| 지금 (Month 1) | 사이트 골격 + 운영 규칙 + 본 전략 spec 완료. 첫 포스트 4가지 후보 중 하나 *느슨하게 draft* 시작 (강제 X) |
| Month 2–3 | 첫 포스트 publish. 캐주얼 톤 self-calibration. 댓글 시스템(utterances) 실제 동작 점검 |
| Month 3–4 | 두 번째 포스트. 누적 태그 모니터링 |
| Month 5–6 | (선택) 세 번째 포스트 + 6개월 retrospective |
| Month 6 review | 페이스 지속 가능? 카테고리 도입 시점? 태그 정리 필요? 실제 경험 기반 재평가 |

### 병행 minor 작업 (블로그 메인 흐름과 별개)

- (선택) `pr_cycle.sh`에 multi-line commit body 지원 추가 — `Co-Authored-By` footer 자동화. `AGENTS.md`에 적은 follow-up
- (선택) layout 미세 보정 — 글 쓰면서 폰트/너비 실제로 거슬릴 때 1줄 PR로 처리

### Anti-goals (이 6개월엔 *안 함*)

- 카테고리 시스템 도입 (글 10편 누적 시점에 재검토)
- 뉴스레터 / 구독자 메일
- 분석 도구 확장 (현재 google-gtag 충분)
- SEO / Open Graph 이미지 / Twitter Card 본격 세팅 (필요시 첫 포스트 직전에 한 줄 PR로)
- 다국어 페이지 본격 지원
- 댓글 시스템(utterances) *사전* 동작 점검을 위한 별도 테스트 포스트 — 첫 진짜 포스트가 자연 trigger

## Out of Scope

- 구체적인 첫 포스트 *콘텐츠 작성* (별도 작업; 본 spec은 *어떤 글을 쓸지의 후보 + 어떻게 쓸지의 규칙*만 정함)
- profile 페이지(About/Experience/Research/Publications) 추가 변경
- 테마 디자인 대규모 변경 (skin 변경, 폰트 패밀리 교체 등)
- 호스팅 / 도메인 / DNS 변경
